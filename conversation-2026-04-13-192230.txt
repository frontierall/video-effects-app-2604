  2. 템플릿 시스템 — videoEffectTemplates.js

  TEMPLATES 배열에 7가지 타입, 총 다수의 템플릿 정의:

  intro / outro / section-title / transition / lower-third / caption

  각 템플릿 구조:
  {
    id, name, type,
    duration: 4000,     // ms
    fields: [...],      // 편집 가능 필드 정의
    audioCues: [...],   // SFX 큐 타이밍 정의
    render(ctx, t, v, W, H) { ... }  // 실제 Canvas 드로잉 함수
  }

  공통 이징 함수: ease, easeOut, easeIn, elasticOut, bounceOut, seededRand(t)
  (글리치용 결정론적 랜덤)

  3. Canvas 렌더러 — useCanvasRenderer.js

  requestAnimationFrame 기반 애니메이션 루프:

  play() → RAF 루프 → renderFrame(t=0~1) → template.render() → setProgress()

  주요 기능:
  - 재생/일시정지/시크 — pausedAtRef로 정확한 위치 보관
  - 이미지 프리로드 — 업로드된 로고를 imageElementRef에 캐시
  - 값 변경 시 즉시 반영 — 재생 중이 아닐 때 현재 위치에서 재렌더

  const samples = useMotionBlur ? 4 : 1;

    const sampleT = t - (i * 0.015) / (duration / 1000);

  5. 오디오 시스템 — videoEffectsAudio.js

  Web Audio API 기반 SFX 7종 프리셋:

  ┌──────────────┬────────────────────────┬───────────────────┐
  │  프리셋 ID   │          종류          │       특성        │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ whoosh_intro │ sawtooth + bandpass    │ 900→240Hz, 0.42s  │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ whoosh_soft  │ sawtooth + bandpass    │ 640→220Hz, 0.32s  │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ click_ui     │ square tone            │ 1400→700Hz, 0.08s │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ impact_hit   │ triangle tone          │ 180→58Hz, 0.22s   │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ pop_accent   │ sine tone              │ 420→620Hz, 0.14s  │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ glitch_zap   │ white noise + highpass │ 1200Hz+, 0.16s    │
  ├──────────────┼────────────────────────┼───────────────────┤
  │ sweep_out    │ sawtooth + bandpass    │ 520→120Hz, 0.38s  │
  └──────────────┴────────────────────────┴───────────────────┘

  미리보기 vs 내보내기 두 가지 컨텍스트:
  - 미리보기: audioContext.destination으로 직접 재생
  - 내보내기: createMediaStreamDestination()으로 스트림 추출 → 비디오와 믹싱

  AI 커스텀 SFX: /api/sfx-generate 호출 → base64 오디오 반환 → customSfxMap에
  저장 → 프리셋 대신 사용

  ---
  6. 내보내기 — useMediaExport.js

  4단계 파이프라인:

  1. 이미지 프리로드
  2. 오프스크린 Canvas (1920×1080 등) + MediaRecorder 생성
  3. RAF 루프로 프레임 렌더 → canvas.captureStream(30fps) → webm 녹화
  4. FFmpeg.wasm으로 webm → H.264 MP4 변환 (Adobe Premiere 호환)

  해상도: 1080p(1920×1080), 720p(1280×720), 480p(854×480)
  비트레이트: 12Mbps 고화질
  오디오: AAC 192kbps
  FFmpeg 실패 시 webm 파일로 폴백

  ---
  다른 컴퓨터에서 이어가는 방법

  1. 코드 받기

  git clone https://github.com/frontierall/trendtube-2603.git
  cd trendtube-2603
  npm install

  2. 환경 변수 설정

  .env 파일을 새 컴퓨터에 직접 만들어야 합니다 (git에 올라가지 않음):
  YOUTUBE_API_KEY=...
  ANTHROPIC_API_KEY=...
  GEMINI_API_KEY=...
  OPENAI_API_KEY=...
  TOGETHER_API_KEY=...

  3. Claude Code 실행

  claude

  4. 메모리는 자동으로 불러옵니다

  Claude Code가 시작되면 방금 저장한 메모리를 자동으로 읽어서 오늘 작업한 내용, 
  미완료 항목, 전체 구조를 그대로 파악합니다.

  ---

  ▎ 단, 메모리는 이 맥의 ~/.claude/ 폴더에 저장되어 있어서 다른 컴퓨터로 자동 
  ▎ 동기화되지 않습니다.
  ▎ 새 컴퓨터의 Claude Code는 처음엔 빈 메모리로 시작하지만, 첫 대화에서 
  ▎ "지금까지 작업 내용 요약해줘" 라고 하면 git 히스토리와 코드를 분석해서 
  ▎ 빠르게 파악할 수 있습니다.
