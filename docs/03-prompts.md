# Prompt Guide

이 프로젝트에는 두 종류의 프롬프트가 있다.

- 사용자가 입력하는 프롬프트
- 서버가 AI 모델에 전달하는 시스템 프롬프트

## 1. 사용자 프롬프트 예시

현재 UI에서 테스트하기 좋은 예시:

- `빨간 파티클이 흩날리며 채널명이 등장하는 인트로`
- `글리치 효과로 화면이 깨지며 전환되는 트랜지션`
- `형광펜이 그어지며 강조 텍스트가 나타나는 자막`
- `시네마틱 레터박스와 함께 에피소드 제목이 공개되는 인트로`
- `소셜 미디어 아이콘이 튀어나오는 아웃트로`

좋은 프롬프트 특징:

- 효과 유형이 명확하다
- 등장 요소가 구체적이다
- 애니메이션 방향이 드러난다
- 색감/톤을 포함하면 결과가 안정적이다

예:

```text
어두운 배경 위에 붉은 네온 라인이 좌우로 스캔되며, 중앙에 채널명이 확대 등장하는 4초짜리 인트로
```

## 2. AI 시스템 프롬프트 구조

핵심 시스템 프롬프트는 [api/ai-template-gen.js](/mnt/d/github_frontierall/video-effects-app-2604/api/ai-template-gen.js:27)에 있다.

주요 제약:

- `function render(ctx, t, v, W, H)` 시그니처 강제
- 첫 줄 배경 초기화 강제
- 마지막 `ctx.globalAlpha = 1` 강제
- 폰트 크기 상대값 사용 강제
- `document`, `window`, `fetch`, `eval`, `import`, `require` 금지

이 프롬프트가 중요한 이유:

- Canvas 코드 생성 품질 확보
- unsafe API 접근 억제
- 정적 템플릿과 같은 형태 유지

## 3. 재사용 가능한 AI 생성 프롬프트 템플릿

```text
{분위기} 톤으로, {주요 요소}가 {등장 방식}으로 나타나고, 마지막에 {마무리 효과}가 들어가는 {효과 유형}.
배경색은 {색상}, 포인트 색상은 {색상}, 전체 길이는 {초}초 느낌으로 만들어줘.
```

예:

```text
시네마틱 톤으로, 에피소드 제목이 중앙에서 천천히 드러나고, 마지막에 부드럽게 페이드아웃되는 인트로.
배경색은 검정, 포인트 색상은 금색, 전체 길이는 5초 느낌으로 만들어줘.
```

## 4. 디버깅용 프롬프트

AI 품질을 확인할 때는 의도적으로 단순한 프롬프트도 필요하다.

- `흰 배경 위에 검은 텍스트가 페이드인되는 심플한 자막`
- `파란 배경에 제목만 중앙에서 확대되는 3초 인트로`

이런 프롬프트는 모델 문제인지, 렌더 코드 문제인지 구분하기 좋다.

## 5. 모델 테스트용 프롬프트

모델별 차이를 비교할 때는 같은 프롬프트를 그대로 써야 한다.

추천 비교 프롬프트:

```text
검은 배경 위에 붉은 선이 좌에서 우로 지나가며, 중앙에 채널명이 타이포그래피로 강하게 등장하는 4초 인트로
```

비교 항목:

- 생성 속도
- JSON 파싱 안정성
- renderCode 길이
- Canvas 코드 품질
- 실제 렌더 결과
