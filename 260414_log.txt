› 인트로 선택시에 예시를 인트로만으로 추천, 아웃트로 선택시에 아웃트로 예시를 5-6

  예시 문구를 준비해 주면 어떨까?


• 좋습니다. 이 방식이 훨씬 초보자 친화적입니다. 지금처럼 한 번에 섞어서 추천하는 것
  보다, 선택한 타입만 집중 추천하고 마음에 안 들면 다시 추천으로 돌리는 쪽이 UX가  
  더 좋습니다.                                                                     
                                                                                   
  정리하면 구조는 이렇게 가는 게 맞습니다.                                         
                                                                                   
  | 선택한 타입 | 보여줄 추천 |                                                    
  |---|---|                                                                        
  | 인트로 | 인트로 예시만 5~6개 |                                                 
  | 아웃트로 | 아웃트로 예시만 5~6개 |                                             
  | 섹션 타이틀 | 섹션 타이틀 예시만 5~6개 |                                       
                                                                                   
  그리고 버튼을 하나 둡니다.                                                       
                                                                                   
  - 예시 다시 추천                                                                 
                                                                                   
  이 버튼을 누르면:                                                                
                                                                                   
  - 같은 카테고리 안에서                                                           
  - 이전 문구와 겹치지 않도록                                                      
  - 새로운 5~6개를 다시 제안                                                       
                                                                                   
  이 구조가 좋은 이유는 명확합니다.                                                
                                                                                   
  1. 사용자가 한 번에 너무 많은 선택지를 보지 않게 됨                              
  2. 원하는 목적에 맞는 예시만 볼 수 있음                                          
  3. “이건 별로네, 다른 느낌 보여줘”가 자연스러움                                  
  4. AI가 추천보다 탐색 도구처럼 느껴져서 만족도가 높아짐                          
                                                                                   
  20가지 예시 문구 풀을 미리 준비하는 것도 좋습니다. 이건 아주 실용적입니다. 다만  
  전부 한 번에 보여주기보다, 내부적으로는 20개 정도의 후보 풀을 가지고 있고, 화면에  는 매번 5~6개씩만 보여주는 방식이 가장 좋습니다.                                 
                                                                                   
  예를 들면:                                                                       
                                                                                   
  - 인트로 문구 풀 20개                                                            
  - 아웃트로 문구 풀 20개                                                          
  - 섹션 타이틀 문구 풀 20개                                                       
                                                                                   
  그리고 화면에는:                                                                 
                                                                                   
  - 1차 추천: 그중 5개                                                             
  - 다시 추천: 남은 것 중 5개                                                      
  - 다시 추천: 또 다른 5개                                                         
                                                                                   
  이렇게 돌리는 방식입니다.                                                        
                                                                                   
  추천 구현 방식도 2단으로 나누면 좋습니다.                                        
                                                                                   
  1. 기본 예시 풀                                                                  
     코드에 미리 준비된 20개 문구 세트                                             
     장점: 빠름, 안정적, 비용 없음                                                 
  2. AI 재추천                                                                     
     기본 예시가 마음에 안 들면 AI가 대본/톤 기반으로 새로운 문구 생성             
     장점: 더 유연함, 문맥 반영 가능                                               
                                                                                   
  즉 처음에는 빠르게 로컬 예시 5~6개를 보여주고, 그 다음 AI로 새 예시 추천 버튼을  
  누르면 완전히 다른 문구를 생성하게 하면 좋습니다.                                

  - 효과 종류 선택
      - 인트로
      - 아웃트로
      - 섹션 타이틀
  - 예시 추천 보기
  - 예시 카드 5~6개
  - 다시 추천
  - 이 예시로 생성

  제가 추천하는 최종 방향은 이겁니다.

  1. 카테고리별 예시 풀 20개씩 준비
  2. 한 번에 5~6개만 노출
  3. 다시 추천 시 이전과 겹치지 않게 교체
  4. 로컬 예시가 부족하면 AI 재추천으로 확장
  5. 선택한 예시를 바로 최종 프롬프트로 연결

  - 인트로 20개
  - 아웃트로 20개
  - 섹션 타이틀 20개
